  11..  NNAAMMEE

  ffiilllluupp - fill one configuration file into another

  22..  SSYYNNOOPPSSIISS

  ffiilllluupp options basefile additionalfile

  Options:

  [[--hheeqqssttvvVV]] [[--dd cchhaarr]] [[--cc cchhaarr]] [[--oo ffiilleennaammee]]

  33..  DDEESSCCRRIIPPTTIIOONN

  This manual page documents the S.u.S.E. program ffiilllluupp.  ffiilllluupp takes
  at least 2 arguments: a basefile and a newfile.  Both of them contain
  e.g. configuration information expressed by e.g. environment variables
  consisting of a variable name, an assignment delimitor and a variable
  value.  ffiilllluupp reads both files and looks up wether variables in the
  new file are not contained in the basefile. New variables are appended
  to the basefile including comments preceeding the variable definition,
  the variable name, the assignment delimitor and the variables value.
  Anything between two variable statements is understood as comment even
  if not introduced by a comment marker.  Logical lines connected by `\'
  are copied as one line. No characters may follow the line connector
  except white space or new line chars.

  The default delimitor is `=', the default comment marker is `#'. Both
  may be altered by command line options.

  While processing the files, ffiilllluupp will display the name of every new
  variable found in newfile. If ffiilllluupp finds duplicate variables in the
  basefile and the newfile, it will issue a warning.

  The strict parsing behaviour of ffiilllluupp concerning the end of file
  condition may be released by using the --ee option.

  44..  OOPPTTIIOONNSS

     --cc cchhaarr,, ----cchhaarr cchhaarr
        use char as comment marker instead of default comment marker
        `#'.
     --dd cchhaarr,, ----ddeelliimm cchhaarr
        use char as assignment delimitor instead of the default
        delimitor `='.

     --ee,, ----iiggnnoorreeeeooff
        ignore eof where it would not be allowed otherwise (i.e. in
        variable names or in variable values).

     --hh,, ----hheellpp
        help: display a short message how to use ffiilllluupp.

     --oo ffiilleennaammee,, ----oouuttppuutt ffiilleennaammee
        do not append output to basefile but write output to filename
        instead. Previous contents of filename will be deleted!

     --qq,, ----qquuiieett
        quiet: no output to screen except errors and warnings.

     --ss,, ----ssuupppprreessss
        suppress comments: do not copy the comments contained in newfile
        to the target file, but only variables and their values.

     --tt,, ----ttrraaiilliinngg
        If there are comment lines at the end of the read file which are
        not followed by a variable statement (kind of premature end of
        file), then transfer that comment to outfile anyway

     --vv,, ----vveerrbboossee
        display any new variable on the screen while processing
        including assignment delimitor and variable value (stdout may be
        redirected)

     --VV,, ----vveerrssiioonn
        print the version of this ffiilllluupp binary and exit with code 0.

  55..  EEXXIITT CCOODDEE

  ffiilllluupp returns -1 on any error.  on successful completion, ffiilllluupp
  returns the (non-negative) number of new variable lines added /
  written to the output file.

  66..  HHIINNTT

  Use -t -d ":" for /etc/passwd or /etc/groups

  Use -d "=" for shell scripts (default)

  To prevent the shell of interpreting your specified delimitor or
  comment marker you better quote those characters as in the examples!
